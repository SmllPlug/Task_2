# Task_2" 
## API-автотесты сервиса Stellar Burgers

Проект содержит набор API-автотестов для проверки основных бизнес-сценариев сервиса Stellar Burgers.

Тестируемый сервис:
https://stellarburgers.education-services.ru/

Тесты написаны на языке Python с использованием pytest и библиотеки requests.  
В проекте используются фикстуры pytest для создания и удаления тестовых данных, а также формируется отчётность в Allure.

# Описание реализованных тестов

## Создание пользователя (TestCreateUser)

Ручка:

POST /api/auth/register

Реализованные проверки:

- Успешное создание уникального пользователя (200)
- В ответе возвращается success: true
- Нельзя создать пользователя с уже существующим email (403)
- Проверяется сообщение "User already exists"
- Ошибка при отсутствии email (403)
- Ошибка при отсутствии password (403)
- Ошибка при отсутствии name (403)
- Проверяется сообщение "Email, password and name are required fields"

Все тесты атомарные — каждый тест проверяет одно состояние системы.

---

## Авторизация пользователя (TestLoginUser)

Ручка:

POST /api/auth/login

Реализованные проверки:

- Успешный логин существующего пользователя (200)
- В ответе возвращается success: true
- Ошибка при неверном email (401)
- Ошибка при неверном password (401)
- Проверяется сообщение "email or password are incorrect"

---

## Изменение данных пользователя (TestChangeUserData)

Ручка:

PATCH /api/auth/user

Для авторизованного пользователя:

- Изменение email (200)
- Изменение password (200)
- Изменение name (200)
- В ответе возвращается success: true
- Нельзя изменить email на уже существующий (403)
- Проверяется сообщение "User with such email already exists"

Для неавторизованного пользователя:

- Попытка изменить email (401)
- Попытка изменить password (401)
- Попытка изменить name (401)
- Проверяется сообщение "You should be authorised"

---

## Создание заказа (TestCreateOrder)

Ручка:

POST /api/orders

Реализованные проверки:

- Создание заказа авторизованным пользователем (200)
- В ответе возвращается success: true
- Создание заказа неавторизованным пользователем (200)
- Создание заказа без ингредиентов (400)
- Проверяется сообщение "Ingredient ids must be provided"
- Создание заказа с неверным хэшем ингредиентов (500)

---

## Получение заказов пользователя (TestGetOrders)

Ручка:

GET /api/orders

Реализованные проверки:

Для авторизованного пользователя:

- Успешное получение списка заказов (200)
- В ответе возвращается success: true

Для неавторизованного пользователя:

- Ошибка получения заказов (401)
- Проверяется сообщение "You should be authorised"

# Фикстуры

В проекте используются pytest-фикстуры для:

- создания пользователя перед тестом
- получения access_token
- автоматического удаления пользователя после выполнения теста

Фикстуры позволяют:

- изолировать тесты
- избежать зависимостей между тестами
- автоматически очищать тестовые данные

---

# Установка зависимостей

pip install -r requirements.txt

# Запуск тестов

pytest

# Генерация Allure-отчёта

Запуск тестов с генерацией отчёта:

pytest tests --alluredir=allure_results

Просмотр отчёта:

allure serve allure_results
